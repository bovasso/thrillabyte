A) HTML Structure - top

This template has liquid layout, which means it's width is adjustable to your browser window, while having minimum width of 900px, which you can change according to your needs.

All elements are named with appropriate IDs and Classes, so you can easily find what you need to edit in the HTML and CSS.

Header

On the top of the page, there is a div with ID of "header". This holds for the title of the website, the user profile menu and the search bar:

<div id="header">

	<h1><a href="#">ADMINIUM</a></h1>
	
	<form action="" method="post" class="searchform">
		<input type="text" class="text" value="Search..." />
		<input type="submit" class="submit" value="" />
	</form>
		
	<div class="userprofile">
		<ul>
			<li><a href="#"><img src="images/avatar.gif" alt="" /> enstyled</a>
				<ul>
					<li><a href="#">Profile</a></li>
					<li><a href="#">Messages</a></li>
					<li><a href="#">Logout</a></li>
				</ul>
			</li>
		</ul>
	</div>

</div>			<!-- #header ends -->

To make the title/logo to link to your homepage, replace the '#' in the anchor tag of the H1, with your website URL.

The search bar resizes when focused, to make more room for typing and for the quick results dropdown. When you start typing in the search bar, after the 3rd symbol, an AJAX call is made, to a file, returning quick results, which you can generate and cache from your back-end.

The user profile menu holds the links for your profile actions, structured in an unordered list. The width of the user profile menu depends on the width of the profile name.

The main navigation is held in an unordered list with an ID of "nav". The currently active item is in strong tags. This changes the color of the item and puts a pointer.

The icons in the main navigation are PNG images for easy customization. The image files could be found in the images/nav/ folder. You can add/replace icons with any 16x16 PNG images. There are some extra icons provided.

The sub-pages are held in nested unordered lists, like you can see at the "Users" item. You can set these to be opened, only if the parent item is active with your back-end script.

The navigation items supports notification badges. These are held in span tags, as could be seen at the "Support" item.

The status box holds an unordered list, containing links with statuses - green, yellow or red lights. The lights are controlled by the classes of the links.

Content

The main content of the page is held by a div with an ID of "content". It's the right, wide part of the layout. The width of the content is liquid, so it will adjust to the browser window size.

Breadcrumb

At the top of the content, there is a breadcrumb, showing back links, for easier navigation.

<div class="breadcrumb">

	<a href="#">Subsection title</a> » 
	<a href="#">Page title</a>
</div>

Messages

Adminium comes with two types of messages - inline messages and HUD messages.

The inline messages comes in 4 styles - Information, Warning, Success and Error. Their structure is as follows:

	<div class="message errormsg"><p>An error message goes here</p></div>
	<div class="message success"><p>A success message goes here</p></div>
	<div class="message info"><p>An informative message goes here</p></div>
	<div class="message warning"><p>A warning message goes here</p></div>

The HUD messages are designed to show result of an action, like AJAX call for example. To trigger a HUD message, in your JavaScript code write:

hudMsg('success', 'Page loaded successfully', 2000);

Where 'success' is the type of message, 'Page loaded successfully' is the text of your message and the third parameter is the time to stay on screen, in milliseconds (2000 = 2 seconds). You can see demo code in js/custom.js, under "HUD Messages".

You can have any type of HUD messages, you just need to make an image for it. For example if you want to have a type of "beer", just make sure to have an image, named msg_beer.png in your images/ folder.

Progress bars

Adminium comes with 2 sizes of progress bars, each in 4 colors.

<div class="progress_bar small">
	<div class="progress_complete" style="width: 25%;">
		<span>25%</span>
	</div>
</div>

<div class="progress_bar small">
	<div class="progress_complete green" style="width: 50%;">
		<span>50%</span>
	</div>
</div>

<div class="progress_bar">
	<div class="progress_complete red" style="width: 75%;">
		<span>75%</span>
	</div>
</div>

<div class="progress_bar">
	<div class="progress_complete orange" style="width: 100%;">
		<span>100%</span>
	</div>
</div>

The div with class "progress_bar" is the holder of the progress bar. Adding class of "small" to it makes it, well... smaller. The div with class "progress_complete" is the actual progress bar, which shows how much is completed, by the width of the element. By adding a second class of "green", "red" or "orange", you can change the colors of the bar. If you use the bigger bars, you can add a percentage number in the "progress_complete" div.

The progress bars could be animated, as you can see in the demo. The animation is done via JavaScript. You can see the code in js/custom.js, under "Progress bars animations":

// Progress bars animations
$('.progress_complete').click(function() {
	var totalWidth = $(this).parents('.progress_bar').width();
	$(this).animate({ width: totalWidth }, 600, function() {
		$(this).find('span').html('100%');
	});
});

Date picker

There is a handy date picker, to use when working with dates. The date picker is powered by the awesome jQuery Date Input plugin. Basically you can add it to any input field with type of "text", just by adding a class of "date_picker", like so:

	<input type="text" class="text date_picker" />

When the input is clicked, a box pops up, displaying a calendar, from which you can pick your date.

Color picker

The template comes with a neat color picker, powered by the jQuery miniColors plugin. You can add it to any input field with type of "text", just by adding a class of "color_picker", like so:

	<input type="text" class="text color_picker" />

Click on the input brings the color picker up.

Custom file inputs

Adminium comes with a unified custom file inputs:

<label>Custom file input label:</label> 
<input type="file" class="file styled" title="Select file 1" /> 
<input type="file" class="file styled" title="Select file 2" />

JavaScript is overwriting each file input, having a class of "styled". The title of the element will be taken to make the label of the input button. When a file is selected, a filed will appear next to the input button, showing the name of the selected file.

On/Off switch

You can use the custom On/Off switch with the following code:

<p class="onoffswitch">
	<input type="checkbox" class="onoffbtn" checked="checked" />
	<label>Checkbox label</label>
</p>

It takes a checkbox and turns it to a switch. If there is a "checked" attribute, the switch will be set to "on", otherwise - to "off". The switching is done by a neat animation.

Sliders

You can have multiple sliders on one page. The sliders could be with single value - with one handle, or ranged - with two handles.

<div class="slider" style="width: 50%;"></div>

<div class="slider-range" style="width: 99%;"></div>

By using class "slider", the result will be a single value slider.
By using class "slider-range", the result will be a slider with two handles, defining a range of values.

You can define the width of your sliders individually, via the width style attribute. For full-width sliders, use width of 99%, so the tooltip, won't make a horizontal scrollbar.

Modal boxes

You can use the modal boxes for displaying remote files. To make a link to open a modal box, you just have to add a class="modal":

	<a href="remote_file.html" class="modal">open modal</a>

The modal boxes are powered by the awesome Fancybox plugin. For full documentation of Fancybox, please visit their site.

Tooltips

The tooltip plugin reads the title attribute of the element and makes a fancy title from it. You can attach the tooltip plugin to any element, by adding a class of "tt" to it.

	<div class="tt" title="This is the tooltip text">Tooltip demo</div>

Sortable image lists with view/delete menus

Adminium gives you a super cool way to manage images - you can sort them via drag & drop, view and delete them. See it at the end of the demo page. Here's how it's done:

<ul class="imglist">
	<li>
		<img src="images/thumb1.jpg" alt="" />
		<ul>
			<li class="view"><a href="big_image.jpg" class="modal" rel="gallery" title="Image title 1">View</a></li>
			<li class="delete"><a href="#deleted">Delete</a></li>
		</ul>
	</li>
	
	<li>
		<img src="images/thumb2.jpg" alt="" />
		<ul>
			<li class="view"><a href="big_image.jpg" class="modal" rel="gallery" title="Image title 2">View</a></li>
			<li class="delete"><a href="#">Delete</a></li>
		</ul>
	</li>
	
	<li>
		<img src="images/thumb3.jpg" alt="" />
		<ul>
			<li class="view"><a href="big_image.jpg" class="modal" rel="gallery" title="Image title 3">View</a></li>
			<li class="delete"><a href="#deleted">Delete</a></li>
		</ul>
	</li>
	
	...
	
</ul>

There is an unordered list with a class of "imglist", each list items contains a thumbnail (100px x 100px) and another unordered list, which contains the "View" and "Delete" links.

Each element is draggable for easy sorting. You can see the many options for the drag & drop at the jQuery UI documentation.

The "View" link will open your full size image in a modal box. You should put the address of your full size image file in the anchor tag.

The "Delete" links could make an AJAX call to delete the file and give the user a message or just follow a regular link. When clicked, the user would be prompted for confirmation "Are you sure you want to delete this image?". If the answer is "Yes", the link will be followed, otherwise nothing will happen. You can customize the question asked, by editing the "Image delete confirmation", in the js/custom.js file.

B) CSS Files and Structure - top

Adminium uses some CSS3 features like rounded corners, text shadows, box shadows, etc. Some of these are still not supported in Internet Explorer, but will degrade gracefully.

The templates comes with these 9 CSS files:

  • style.css contains the major styles, layouts, etc.
  • green.css contains the styles for the green skin.
  • red.css contains the styles for the red
  • ie.css contains fixes for IE7 and IE6.
  • jquery.wysiwyg.css is the CSS file for the WYSIWYG plugin, which turns a textarea into rich text editor.
  • wysiwyg.css contains custom styles for the WYSIWYG editor.
  • fancybox.css is the CSS file for the Fancybox plugin, used for modal boxes.
  • visualize.css is the CSS file for the jQuery Visualize plugin, that creates the graphs.
  • date_input.css - the CSS that styles the calendar box for the date picker
  • jquery.minicolors.css CSS for the color picker

The style.css is separated into logical sections:

	
/* Header */ - The rules, formatting the header

/* User menu */ - The styled for the user menu

/* Search form */ - The search bar in the header and the quick results

/* Sidebar */ - The styles for the sidebar, the navigation and the status box

/* Content */ - The styles for the content

/* Statistics */ - The styles for today's statistics. If you need to edit the statistics graph CSS, yo can find it in css/visualize.css

/* Messages */ - Inline messages (info, success, warning, error)

/* HUD Messages */ - HUD messages styles

/* Progress bars */ - Progress bars

/* Forms */ - Styles for the form elements

/* Image list */ - The image management mini menus styles

/* Slider */ - The styles for the sliders

/* Login page */ - The styles for the login page

Skins

By default the template will use the original, blue skin. If you want to use the green or red skin, you should include green.css or red.css in the head of the page:

<style type="text/css" media="all">
@import url("css/style.css");
@import url("css/visualize.css");
@import url("css/date_input.css");
@import url("css/jquery.minicolors.css");
@import url("css/jquery.wysiwyg.css");
@import url("css/jquery.fancybox.css");
@import url("css/tipsy.css");

@import url("css/green.css");
</style>

Make sure to include it as the last file, since the skins overwrite some of the styles from the other files.

C) JavaScript - top

The template uses jQuery for enhancements and effects. You should find these files in the /js/ folder:

  1. jquery.wysiwyg.js the plugin which turns a textarea into rich text editor
  2. jquery.date_input.min.js is the date picker plugin
  3. jquery.minicolors.min.js - the color picker plugin
  4. jquery.fancybox.js is the plugin, used for modal boxes
  5. excanvas.js is a plugin that adds canvas support to Internet Explorer 6
  6. jquery.visualize.js is the plugin that creates the graphs
  7. jquery.tablesorter.min.js the plugin for sortable tables
  8. jquery.tipsy.js the plugin for the tooltips
  9. custom.js is the file containing most of the settings for the plugins and some custom scripts

The jQuery core and jQuery UI files are loaded from the official jQuery CDN, as best practices recommends. You can always download them and include them locally, if needed.

D) PSD Files - top

In the /psds folder, you can find three .psd files:

  1. mbg.psd - Contains the color gradients for the 3 skins. All the button inputs and the navigation hover uses this background.
  2. onoff.psd - The graphic for the On/Off switch
  3. handle.psd - The graphic for the slider handles
